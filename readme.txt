Gulp セットアップおよび使用方法
このガイドは、プロジェクトでGulpをセットアップして実行する方法を説明しています。以下の手順に従って、Gulpをローカル環境で動作させてください。

前提条件
Gulpを使うために、以下のソフトウェアがインストールされていることを確認してください。

Node.js と npm (Node Package Manager)

Node.jsのダウンロード からインストールしてください。npmはNode.jsに同梱されています。
Gulp CLI (コマンドラインインターフェース)

Gulp CLIをグローバルにインストールするために、以下のコマンドをターミナルで実行します。
npm install --global gulp-cli
プロジェクトセットアップ
前提条件をインストールしたら、以下の手順でプロジェクトをセットアップします。

リポジトリをクローンする (まだしていない場合):

git clone <リポジトリURL>
プロジェクトディレクトリに移動する:

cd <プロジェクトディレクトリ>
プロジェクト依存パッケージをインストールする:

以下のコマンドを実行して、Gulpやその他必要なnpmパッケージをインストールします。

npm install
Gulpタスクの実行
セットアップが完了したら、以下のGulpタスクを実行できます。

1. Gulpの起動 (デフォルトタスク)
このタスクは、SCSS、JavaScript、画像ファイルの変更を監視し、自動的にコンパイル、ミニファイ（圧縮）、およびリアルタイムでの更新を行います。

gulp
このコマンドにより以下が実行されます：

SCSSのコンパイル (style.cssの生成)
コンパイルされたCSSのミニファイ（圧縮）(style.min.cssの生成)
JavaScriptファイルのミニファイ（圧縮）とmin/ディレクトリへの出力
画像の最適化とmin/ディレクトリへの出力
ファイルの変更が検出されると自動でブラウザをリロード（ローカル開発サーバーが必要です。例えば、Localなど）。
2. 手動での画像最適化
Gulpのデフォルトタスクを実行せずに、画像の最適化だけを行いたい場合は、以下のコマンドを実行します。

gulp images
これにより、すべての画像が最適化され、min/ディレクトリに出力されます。

フォルダ構成
Gulpに関連する主要なファイルやフォルダの構造は以下の通りです。

/scss          - SCSSファイル
/css           - コンパイルされたCSSファイル (style.cssおよびstyle.min.css)
/js            - JavaScriptファイル
/js/min        - ミニファイされたJavaScriptファイル
/images        - 元の画像ファイル
/images/min    - 最適化された画像ファイル

よくある問題
Gulpコマンドが見つからない: ターミナルで npm install --global gulp-cli を実行して、Gulp CLIがグローバルにインストールされているか確認してください。
依存関係が見つからない: プロジェクトディレクトリで npm install を実行し、すべての必要なパッケージがインストールされていることを確認してください。
このガイドに従えば、初心者でもGulpを使った開発環境を簡単にセットアップできるはずです。
